# Example Patterns

Use these patterns to choose structure and clause emphasis quickly.

## Pattern 1: Enterprise WeChat professional edition + private-domain service

Typical signals:

- enterprise WeChat professional edition
- six online modules / feature list
-陪跑,运营 support, customer retention, traffic, group management
- account protection or account status caveat

Drafting moves:

- use a service cooperation contract
- place the feature list into the service scope section or an annex
- state the service term clearly
- separate included service from excluded account unblocking
- include one training / onboarding clause if the facts support it
- define acceptance around module availability, configuration completion, and data migration or tool enablement

## Pattern 2: Tool authorization + install fee + operation quota + overage

Typical signals:

- professional tool fee plus one-time install fee
- monthly user / traffic / interface quota
- overage pricing
- fixed service term such as three months

Drafting moves:

- use a tool service cooperation contract
- write the base package, quota cap, and overage unit price separately
- define whether the monthly volume is a service configuration standard or a guaranteed outcome
- require written notice and payment before extra interfaces or extra capacity are activated
- make termination and data handover clauses explicit

## Pattern 3: Overseas market / Korea / channel / official title cooperation

Typical signals:

- market exclusivity
- channel ownership or lead routing
- overseas general manager or official partner naming
- settlement principles to be decided later

Drafting moves:

- use a cooperation memo or framework memo unless pricing and execution are already fixed
- define the responsible person, market scope, exclusivity boundary, and no-bypass rule
- limit official title usage to written authorization scope
- reserve revenue settlement and later legal-entity arrangements for follow-up written agreements
- add confidentiality and dispute clauses, but keep execution obligations at principle level

## Pattern 4: AI / software / agent / POC / formal deployment

Typical signals:

- POC quote
- formal deployment after testing
- API integration, model routing, maintenance, optional modules
- web front end, report generation, workflow, deployment

Drafting moves:

- use a development and deployment cooperation contract
- split the project into POC phase and formal phase
- mirror quote-line items in scope and payment clauses
- define testing, acceptance, remediation, optional modules, and运维 response
- keep platform IP and base technology with乙方 unless the deal clearly transfers them
- give甲方 agreed usage rights to project results and generated business data

## Pattern 5: Prior proposal already exists in the chat

Typical signals:

- the latest message is short, but earlier messages already contain a方案 or报价
- screenshots or quote tables describe the real scope

Drafting moves:

- do not draft from the latest message alone
- pull the earlier plan into the contract objective, scope, milestones, and exclusions
- if the quote is detailed, use a table or annex instead of compressing it into one vague paragraph

## Pattern 6: Missing corporate details

Typical signals:

- only company names are given
- no address, unified social credit code, bank account, or legal representative

Drafting moves:

- keep the contract usable by leaving formal identity fields blank
- never fabricate registration details
- fill only the bank information or tax numbers explicitly supplied by the user
