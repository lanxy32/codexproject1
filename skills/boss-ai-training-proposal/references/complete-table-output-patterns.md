# Complete Table Output Patterns

Use this reference when the user expects the proposal to show full tables and real row data rather than placeholders.

This is especially important when:

- the user references an attached proposal or training document,
- the prior version of the scheme used placeholder text like `表格` or `点击图片查看`,
- the proposal includes monthly plans, quarterly plans, curriculum matrices, or time-based schedules.

## Core rule

If a table is logically needed, render it fully in markdown.

Do not say:

- `表格`
- `点击图片可查看完整电子表格`
- `见附件表格`
- `此处略`

unless the user explicitly asks for a short summary only.

## When to use full tables

Render full tables when the proposal includes:

- 月度线上课安排
- 季度线下课安排
- 2天或3天培训排期
- 课程菜单
- 服务流程
- 交付清单

## Table quality checklist

A good table:

- has a clear header row
- has complete rows, not only one example row
- uses business-useful columns
- does not hide key data in prose below
- can be copied directly into Word, PPT, or Feishu docs

## Common table shapes

### 1. 服务流程表

| 阶段 | 服务内容 | 目标 |
|---|---|---|
| 需求调研阶段 | 问卷、访谈、场景摸底 | 明确培训重点 |
| 培训实施阶段 | 线上或线下课程执行 | 完成认知与实操 |
| 跟进支持阶段 | 答疑、复盘、模板沉淀 | 保证落地 |

### 2. 月度线上课表

| 期次 | 频率 | 时长 | 培训主题 | 重点内容 | 适合对象 | 课后输出 |
|---|---|---|---|---|---|---|
| 第1期 | 每月1次 | 2小时 | AI基础工具入门 | 工具认知、基础提示词 | 全员 | 基础模板 |
| 第2期 | 每月1次 | 2小时 | AI办公自动化 | 文档、纪要、总结 | 全员 | 办公模板 |
| 第3期 | 每月1次 | 2小时 | AI宣传辅助 | 宣传文案、海报辅助 | 宣传/运营 | 宣传模板 |

### 3. 季度线下课表

| 期次 | 频率 | 时长 | 培训模块 | 重点内容 | 重点对象 | 关键产出 |
|---|---|---|---|---|---|---|
| 第1期 | 每季度1次 | 2天 | AI办公与宣传实操 | 文档、PPT、宣传内容、岗位实操 | 骨干员工 | 实操作业 |
| 第2期 | 每季度1次 | 2天 | AI岗位场景深化 | 客服、运营、行政高频任务 | 重点岗位 | 岗位模板 |

### 4. 时间排期表

| 序号 | 时间 | 模块 | 课程主题 | 课程详细内容 | 关键产出 |
|---|---|---|---|---|---|
| 1 | 09:30-10:20 | AI认知 | 工具与应用全景 | 常用工具、使用边界、企业场景 | 统一认知 |
| 2 | 10:20-11:10 | 提示词 | 高效提问方法 | 结构化提示词与模板 | 提示词模板 |

## Use with attachment-derived examples

If the attachment shows a stronger table style:

- borrow its columns,
- borrow its level of detail,
- materialize the rows directly in the new answer.

Do not mention that the source file had hidden or image-based tables. Your job is to make the new output complete.
