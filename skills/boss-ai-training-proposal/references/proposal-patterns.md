# Proposal Patterns

Use this reference to turn rough老板需求 into a commercial but realistic AI内训方案.

## Best-practice engagement design

Recommend a closed loop instead of a one-off class:

1. 需求调研
2. 初案调整
3. 现场内训
4. 部门场景与智能体清单梳理
5. 8周线上深化学习
6. 1个月线上咨询
7. 工具选型建议与后续建设建议

This gives the boss a stronger reason to buy because it connects training with implementation support and creates a clear post-training learning path.

## Three-day framework

### Day 1: shared foundation for all participating departments

Focus on common language and tool habits:

- AI认知统一 and what AI can or cannot replace
- Common prompt habits and reusable prompt structure
- Safe use boundaries, especially for company data
- Common writing, summarization, SOP, and meeting-note tasks
- On-site梳理 of department scenario lists and candidate agent opportunities

Goal:
make every department see a clear first use case on the same day.

Recommended client-facing output shape:

| 天数 | 培训对象 | 培训模块 | 培训内容 | 关键产出 |
|---|---|---|---|---|
| 第一天 | 全部门 | AI认知统一 + 场景梳理 | 工具认知、提示词基础、岗位场景盘点、智能体清单梳理 | 场景清单初稿 |

### Day 2: growth and content workflow

Use for front-office creative and traffic teams:

- 设计部
- 文案策划部
- 拍摄剪辑部
- 直播切片部
- 直播运营部
- 推流部

Focus:

- content ideation
- script and shot-list generation
- visual and short-video asset assistance
- live-room script and review packs
- material reuse and batch production

Recommended client-facing row title:
`内容、直播、设计类部门专项实操`

### Day 3: service, operations, and organization workflow

Use for support teams:

- 客服部
- 采购部
- 仓储部
- 人事部

Focus:

- FAQ and SOP assistants
- standard summaries and reports
- comparison tables and decision support
- onboarding, policy, and internal document drafting
- exception review and process standardization

Recommended client-facing row title:
`供应链、客服、人事类部门专项实操 + 智能体落地复盘`

## Proposal wording patterns

Prefer wording like:

- `帮助客户建立AI应用共识与落地方法`
- `围绕真实岗位场景完成实操训练`
- `先梳理场景清单，再确定工具搭建路径`
- `以高频提效任务为切入口，逐步延展到流程协同与智能体建设`
- `培训后通过线上咨询和每周学习机制确保落地`

Avoid wording like:

- `三天内全面完成企业AI转型`
- `所有岗位都能立即用智能体替代`
- `一套工具解决所有部门问题`

## Scope boundaries

State boundaries clearly when helpful:

- Exclude finance or highly sensitive data if the client requests it
- Do not promise private deployment unless the client asks for it
- Do not force advanced agent building into day 1 if the company is 0基础
- Separate `现场培训目标` from `后续系统建设目标`

## Minimal proposal skeleton

```md
项目概述
- 客户当前AI基础
- 部门覆盖范围
- 当前核心痛点

项目目标
- 提升效率
- 提升标准化
- 沉淀可复用方法

整体服务流程
- 需求调研
- 3天集中内训
- 8周线上深化
- 1个月线上咨询

课程安排
- 第一天：全部门统一认知与通用能力
- 第二天：增长与内容相关部门
- 第三天：客服、供应链与组织支持部门

交付成果
- 培训大纲
- 场景清单
- 智能体清单初稿
- 8周学习安排表

后续服务
- 学习资料包
- 1个月线上咨询
```

## Do not do this

- Do not answer image-style requests with only prose.
- Do not leave placeholders like `表格暂时无法展示`.
- Do not produce over-broad titles that fail to map to actual department training needs.
