# Curriculum Frameworks

Use this reference when the user wants the output to look like the attached course images: structured, layered, table-first, and suitable for proposals or PPT pages.

## 1. AI培训课程清单大纲

Use this when the client asks for a top-level课程矩阵 or wants a professional training menu.

Recommended columns:

| 天数 | 级别 | 模块 | 培训内容 | 课程详细内容 | 费用 |
|---|---|---|---|---|---|
| 1-2天 | 初阶科普 | 大模型基础 | 初识大模型 | AI大模型基本概念、发展趋势、企业应用方向 | 按需填写 |

Default level ladder:

- `初阶科普`: AI认知、工具认知、提示词基础、常见办公应用
- `中阶岗位实操`: 文案、客服、营销、运营、设计、数据分析
- `高阶智能体研修`: 知识库、工作流、插件、流程编排、智能体搭建

When adapting for a 3-day客户方案:

- Day 1 maps to `初阶科普`
- Day 2 maps to `中阶岗位实操`
- Day 3 maps to `中阶岗位实操 + 高阶智能体入门`

## 3. 3天内训排期表

Use this when the user wants a client-facing implementation outline.

Recommended columns:

| 天数 | 培训对象 | 培训模块 | 培训内容 | 关键产出 |
|---|---|---|---|---|
| 第一天 | 全部门 | AI认知与应用准备 | 工具安装、提示词基础、应用场景盘点、智能体清单梳理 | 部门应用清单初稿 |

If the user wants a more formal教案, upgrade the table to:

| 序号 | 天数 | 时间 | 培训模块 | 课程主题 | 课程详细内容 | 关键产出 |
|---|---|---|---|---|---|---|
| 1 | 第一天 | 09:30-10:20 | AI认知与应用准备 | AI基础认知 | 工具认知、应用边界、企业场景理解 | 统一认知 |

If the user wants a stronger实战方案, keep the table and add module blocks below it:

```md
模块一：<模块名称>（1.5小时）
- 核心目标：<...>
- 主要内容：
  1. <...>
  2. <...>
- 互动形式：<...>
- 关键产出：<...>
```

Recommended logic:

- 第一天：全员统一基础、工具上手、需求梳理、智能体清单盘点
- 第二天：内容/直播/设计链路专项实操
- 第三天：客服/供应链/人事链路实操 + 落地复盘

Use exact time ranges by default. Do not collapse sessions into `上午/下午` unless the user only wants a rough outline.

## 6. Writing rules for image-style outputs

- Use tables first, prose second.
- Keep row titles short and business-like.
- Prefer `培训内容` plus `课程详细内容` over long paragraph explanations.
- Make every row teachable and deliverable; avoid vague titles with no实操内容.
- When the company is 0基础, each day should contain a visible产出, not just understanding.
- If the user asks for a formal proposal, combine `项目概述 + 服务流程 + 表格课程大纲 + 配套服务升级`.
- Prefer concrete session times over `上午/下午`, especially in 2-day and 3-day training tables.
- For higher-quality schemes, add module-level goals, interactions, and outputs after the main table instead of stopping at the schedule alone.
