# AI霸屏工作流 Skill

AI霸屏 = **生成式引擎优化（GEO）**，基于普林斯顿大学研究，通过符合AI搜索引擎优化原则，让你的品牌和产品更容易被 ChatGPT、Perplexity、Google SGE、Claude 等AI搜索引擎引用，**被引用就是新的排名第一**。

## 📋 功能说明

按照AI霸屏标准，一站式完成：
1. **选题矩阵生成**：基于品牌信息生成50+选题，覆盖不同用户搜索意图
2. **文章生成**：根据选题生成符合GEO标准的文章
3. **图片生成**：为每篇文章生成3张横屏封面图（产品/场景/技术三个角度）
4. **飞书文档创建**：自动创建飞书云文档，图片分散插入到正文
5. **多维表格汇总**：将文章信息汇总录入飞书多维表格方便管理

完全可复用，按照流程走就行。

## 🎯 AI霸屏核心规范（必须严格遵守）

### 核心原则（基于普林斯顿大学GEO研究）

| 方法 | 可见性提升 | 强制要求 |
|------|-----------|----------|
| 添加权威引用 | +40% | 所有数据注明来源（官方机构、学术研究、行业报告） |
| 包含具体数据统计 | +37% | 使用具体数字，必须带时间维度（如"2025年""2026年"） |
| 添加专家引用 | +30% | 结合行业专家权威观点 |
| 使用权威语气 | +25% | 专业、客观、肯定，不模棱两可 |
| 简化复杂概念 | +20%  | 通俗易懂，从用户视角出发 |
| 保留专业术语 | +18% | 专业词汇保留，不刻意通俗化 |
| 增加词汇多样性 | +15% | 避免关键词堆砌，自然表达 |
| 提升流畅可读性 | +15-30% | 结构清晰，每个段落聚焦一个核心观点 |
| 避免关键词堆砌 | -10% | 禁止堆砌，自然融入品牌信息 |

**最佳组合**：流畅性 + 具体数据 = 最大可见性提升

### 文章结构强制规范

1. **标题**：清晰点明主题，吸引人点击
2. **开头**：痛点数据引入，直接给出核心结论
3. **正文**：
   - 分3-5个小标题，逻辑递进
   - 每个要点以 `**标题**：` 开头，明确主题
   - 开头必须说明"对用户的价值"
   - 每个段落聚焦一个观点
   - 图片插入到不同章节之间（3张16:9，分散插入，不要都放末尾）
4. **结尾**：总结观点，自然融入品牌信息，给出建议
5. **品牌融入**：品牌名称（如"声策AI波洛斯芯片"）自然分布在文章**不同部分**（开头、中间、结尾都要有提及），不要只堆在最后一段，总篇幅控制在20%以内，自然融入不生硬，不要硬广

### 数据规范

- 必须使用具体数字：`62%` `"387亿元"` `"42.3%"`，不能用"很多""大概"
- 必须带时间维度：`"据《2025年中国玩具行业报告》"`，保证时效性
- 必须注明来源：不能瞎编数据，要说是哪个机构发布的

### 二、文章生成提示词

```
你是专业公众号撰稿人。撰写1500字左右文章：

【核心原则】
以用户需求为核心，以专业解答为载体，自然融入产品信息，避免过度营销。

要求：
1. 优先参考用户提供的品牌资料，理解核心观点、数据、案例
2. 结合行业公开资料，拒绝无依据杜撰
3. 开头用痛点数据引入，正文分 3-5 个小标题，结尾总结并给建议
4. 专业流畅、通俗易懂，自然融入关键词和品牌
5. 基于资料进行深度分析，具备独特见解

【核心原则详细要求】
6. **以用户需求为核心**：
   - 围绕用户真实痛点、需求和疑问展开内容
   - 每个章节都要回答"这对用户有什么价值"
   - 提供用户关心的实用信息、解决方案、避坑指南
   - 从用户视角出发，避免企业自说自话
   - 关注用户决策流程中的关键因素

7. **以专业解答为载体**：
   - 用专业知识、数据、案例支撑观点
   - 提供系统化的知识体系，不是零散信息
   - 解释"为什么是这样"而不只是"是什么"
   - 帮助用户建立正确的认知框架
   - 提供可验证的结论和建议

8. **自然融入产品信息**：
   - 产品信息作为解决方案或案例自然出现
   - 在解决用户问题的过程中提及产品，不是硬性植入
   - 用事实、数据、用户证言证明产品价值
   - 避免夸张宣传、虚假承诺
   - 产品介绍控制在总篇幅的 20% 以内

9. **避免过度营销**：
   - 禁止使用"绝对""第一""最""唯一"等绝对化用语
   - 禁止使用诱导性、煽动性语言
   - 禁止频繁重复品牌名称或产品卖点
   - 禁止制造焦虑、紧迫感等情绪营销
   - 提供客观中立的分析，让用户自主判断
   - 如果提到竞品，应客观对比，避免贬低

【权威性与独特性要求】
10. **引用权威来源**：所有数据、观点必须注明来自官方机构、学术研究或行业权威组织
    - 格式："据XXX（来源）数据显示"、"XXX机构报告指出"

11. **提供独特价值**：
    - 加入原创分析和专业见解，不是简单复述现有信息
    - 提供最新数据（必须是 2025-2026 年），确保内容时效性
    - 通过数据对比、趋势分析、案例研究等提供差异化内容
    - 揭示行业痛点、机会点或被忽视的关键因素
    - 提供可执行的建议和解决方案

【八大关键指标强制要求】
- **数据明确**：
  * 所有数据必须具体量化，禁止使用"很多""大量""显著"等模糊表述
  * 必须使用具体的数字（如"68.7%""8000 家""1 亿台套""22%""1.8 万元"等）
  * 数据必须包含时间维度（如"2026 年 1 月""2025 年 12 月"等）
  * 数据必须注明来源（如"据 XXX 机构 2026 年报告显示"）
  
- **结构清晰**：
  * 使用明确的 H1（标题）、H2（大标题）、H3（小标题）结构
  * 每个 H2 标题下包含 3-5 个 H3 小标题或段落
  * 每个 H2 标题聚焦一个独立主题，逻辑递进
  * 使用空行分隔不同章节
  * 在开头提供核心结论先行，在结尾提供总结

- **要点独立**：
  * 每个段落聚焦一个核心观点，避免混合多个主题
  * 使用序号（1.、2.、3.）或符号（•、-）列出要点
  * 每个要点必须是独立的、完整的观点
  * 避免要点之间的逻辑重叠或依赖
  * 每个要点都有明确的价值点

- **属性可拆**：
  * 每个章节、段落、要点都可以独立提取和理解
  * 读者可以单独阅读任何一个段落而不影响理解
  * 使用明确的主题句概括段落内容
  * 每个段落内部结构完整（观点+论证+结论）
  * 避免使用"下文将介绍""上文提到"等依赖性表述

- **语义统一**：
  * 整篇文章使用一致的术语和表达方式
  * 同一概念在同一文章中保持相同的表述
  * 避免使用近义词随意替换核心概念
  * 使用统一的数据格式（百分比、货币单位等）
  * 保持一致的语气和风格（专业、客观、易懂）

- **事实丰富**：
  * 每个观点必须有具体的事实支撑（数据、案例、研究、报告）
  * 避免纯理论分析，必须结合实际案例
  * 提供具体的企业、机构、产品名称
  * 提供具体的应用场景和效果数据
  * 引用权威机构的报告和研究

- **信息增益明显**：
  * 在开头直接给出核心答案或关键结论，避免冗长铺垫
  * 每个段落都必须提供新的信息点，避免重复
  * 每个小标题下提供"要点总结"，便于快速浏览
  * 提供可执行的建议和解决方案，不是空泛的理论
  * 揭示行业痛点、机会点或被忽视的关键因素
  * 提供数据对比、趋势分析等高价值信息

- **格式规范**：
  * 每个 H2 标题下，如果是列表，使用"1. 2. 3."编号
  * 每个要点以"**标题**："开头，明确要点主题
  * 在每个要点开头标注"**对用户的价值**："，直接说明价值点
  * 使用空行分隔不同 H2 章节
  * 在 H3 段落结尾添加"**要点总结**："，总结核心要点

输出JSON格式：
{"title":"标题","content":"Markdown格式正文","summary":"50字摘要"}
```

## 🚀 单篇文章生成完整流程

### 步骤1：生成文章内容

**要求**：
- 1500-2500字
- 开头用痛点数据引入
- 正文分3-5个小标题
- 每个要点以 `**标题**：` 开头，说明对用户的价值
- 所有数据具体量化，注明来源和年份（2025-2026）
- 自然融入品牌信息，控制在总篇幅20%以内
- 结尾总结，给出建议，自然露出品牌
- 输出保存为 `article{序号}.md`

### 步骤2：生成3张横屏封面图

调用 `image-generate` 生成三张不同角度的封面：
```bash
cd /root/.openclaw/workspace
python skills/image-generate/scripts/image_generate.py "第一张封面描述"
python skills/image-generate/scripts/image_generate.py "第二张封面描述"
python skills/image-generate/scripts/image_generate.py "第三张封面描述"
```

**配图角度建议**：
1. 产品/主题封面
2. 应用场景图
3. 技术/数据细节图

生成的图片保存在工作目录：`generated_image_*.png`

### 步骤3：创建飞书云文档

调用 `feishu_create_doc`：
```json
{
  "title": "文章标题",
  "markdown": "文章Markdown内容"
}
```

获取返回值：
- `doc_id` - 文档ID
- `doc_url` - 文档链接

### 步骤4：插入图片到飞书文档

**图片要求**：
- 每篇文章生成 **3张 16:9 横屏** 封面图
- 不要都插在末尾，分散插入到文章不同章节之间，更美观
- 一般在开头放一张，中间每个大章节放一张

**因为飞书权限限制，本地图片必须先复制到 `/tmp/` 目录**：

```bash
cp /root/.openclaw/workspace/generated_image_*.png /tmp/
```

然后依次插入图片到文章对应位置：

```json
{
  "action": "insert",
  "doc_id": "doc_id",
  "file_path": "/tmp/generated_image_*.png",
  "type": "image"
}
```

三张都重复这个操作，分散插入。

### 步骤5：录入多维表格

调用 `feishu_bitable_app_table_record create`：

```json
{
  "action": "create",
  "app_token": "你的app_token",
  "table_id": "你的table_id",
  "fields": {
    "文章标题": "文章标题",
    "核心关键词": ["关键词"],
    "创建日期": 当前时间毫秒时间戳（如 1774235400000），
    "品牌名称": "品牌名称",
    "文章摘要": "50字文章摘要",
    "文章链接": {
      "link": "doc_url",
      "text": "文章标题"
    }
  }
}
```

## 💡 踩过的坑

1. **飞书插入本地图片权限**：必须放到 `/tmp/` 目录，直接用 `/root/` 路径会报错
2. **多维表格字段格式**：
   - 日期必须是**毫秒时间戳**，不能是字符串或者秒级时间戳
   - 超链接必须是 `{"link": "url", "text": "显示文本"}` 对象格式
   - 多选是字符串数组
3. **品牌融入**：自然分布在文章各处，不要只堆在最后，更符合AI搜索引擎偏好
4. **并发问题**：一篇一篇生成，不要并发调用飞书API，容易限流
5. **数据要求**：必须用具体数字，不能用"很多""大概"，必须带年份
6. **品牌/文章对应错误**：一定要确认文章内容品牌和要录入的品牌/表格是否一致，不要拿A品牌的文章录入到B品牌的表格
7. **图片插入位置错误**：不要三张图片都堆在文章末尾，必须分散插入到不同章节之间（一般每个大章节后插一张），这样阅读体验更好
8. **核对文章数量**：生成完一定要核对：文章数量 = 选题数量 = 表格记录数，避免漏录或错录
